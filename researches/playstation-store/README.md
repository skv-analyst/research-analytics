# PlayStation Store Research

Проект для сбора данных об играх и их деталях с PlayStation Store.

## Структура проекта

```
playstation-store/
├── src/
│   ├── config.py           # Конфигурация и параметры запроса
│   ├── get_games.py        # Сбор списка игр из каталога
│   ├── get_details.py      # Сбор расширённых данных по играм
│   └── get_comments.py     # (опционально) Сбор комментариев
├── data/
│   ├── ps-store-games.csv  # Список собранных игр
│   └── ps-store-games-details.csv  # Расширённые данные по играм
└── pyproject.toml          # Зависимости проекта
```

## Установка зависимостей

```bash
cd researches/playstation-store
uv sync
```

## Сбор данных

### 1. Сбор списка игр (`get_games.py`)

Собирает список всех игр из каталога PlayStation Store для указанного региона.

**Базовый запуск:**
```bash
uv run src/get_games.py
```

**С параметрами:**
```bash
uv run src/get_games.py --country en-pl
```

**Параметры:**
- `--country` — Локаль стора (по умолчанию `en-in`, например: `en-us`, `tr-tr`, `de-de`)

**Результат:**
- Сохраняет список игр в `data/ps-store-games.csv`
- Продолжает сбор с последней сохранённой позиции при повторном запуске

---

### 2. Сбор расширённых данных (`get_details.py`)

Собирает подробную информацию о каждой игре: цены, языки озвучки и субтитров, количество игроков, жанры, рейтинги и т.д.

#### Режимы запуска

**Обычный режим: сбор новых игр для региона**
```bash
uv run src/get_details.py --country en-pl
```
Обрабатывает только игры, которые ещё не были собраны для указанного региона.

---

**Тестовый режим: первые N игр**
```bash
uv run src/get_details.py --country en-in --test --limit 5
```
Собирает данные только по первым 5 играм (полезно для проверки).

**Параметры:**
- `--test` — Включить тестовый режим
- `--limit N` — Количество игр для обработки (по умолчанию 5)

---

**Точечный сбор по одной игре**
```bash
uv run src/get_details.py --country en-pl --product-id UP0006-PPSA19534_00-SANTIAGOSTANDARD
```
Собирает данные только по одной конкретной игре.

**Параметры:**
- `--product-id <ID>` — ID игры для сбора

---

**Повторный сбор только по "битым" записям**
```bash
uv run src/get_details.py --country en-pl --retry-missing
```
Обрабатывает только игры в `ps-store-games-details.csv`, где пусты поля:
- `voice_langs` (языки озвучки)
- `screen_langs` (языки субтитров)
- `offline_players` (количество локальных игроков)

Полезно, если при первом сборе сервер отключил доступ по IP из-за лимитов.

---

**Повторный сбор с увеличенной паузой между запросами**
```bash
uv run src/get_details.py --country en-pl --retry-missing --sleep-seconds 1.5
```
Увеличивает паузу между запросами до 1.5 секунд (по умолчанию 0.3 сек).

**Параметры:**
- `--sleep-seconds <число>` — Пауза в секундах между запросами

---

#### Комбинированные примеры

**Тестирование точечного сбора с увеличенной паузой:**
```bash
uv run src/get_details.py --country en-pl --product-id EP8204-PPSA28507_00-IREMCOLLECT03PS5 --sleep-seconds 1.2
```

**Тестирование режима повторного сбора на первых 10 "битых" записях:**
```bash
uv run src/get_details.py --country en-pl --retry-missing --test --limit 10 --sleep-seconds 1.5
```

**Полный набор параметров для восстановления после блокировки по IP:**
```bash
uv run src/get_details.py --country en-pl --retry-missing --sleep-seconds 2.0
```

## Общие параметры

| Параметр | Описание | По умолчанию |
|----------|---------|--------------|
| `--country` | Локаль стора (en-in, en-pl, tr-tr, de-de, etc.) | en-in |
| `--test` | Тестовый режим (только первые N игр) | выключено |
| `--limit` | Количество игр для обработки в тестовом режиме | 5 |
| `--retry-missing` | Повторный сбор только по "битым" записям | выключено |
| `--product-id` | Собрать данные только по одному ID | не указано |
| `--sleep-seconds` | Пауза между запросами (секунды) | 0.3 |

---

## Конфигурация

Основные параметры находятся в `src/config.py`:

- `API_URL` — URL API GraphQL PlayStation Store
- `HEADERS` — HTTP-заголовки для запросов
- `DETAILS_TIMEOUT_SECONDS` — Таймаут для HTTP-запросов (10 сек)
- `DETAILS_SLEEP_SECONDS` — Стандартная пауза между запросами (0.3 сек)

## Работа с данными

### Структура CSV файлов

**ps-store-games.csv:**
- `id` — Уникальный ID игры в PlayStation Store
- `name` — Название игры
- `platforms` — Платформы (PS4, PS5, и т.д.)
- `offset` — Смещение при последней загрузке

**ps-store-games-details.csv:**
- `product_id` — ID игры
- `country` — Регион (en-in, en-pl, и т.д.)
- `base_price` — Базовая цена
- `discounted_price` — Цена со скидкой
- `genres` — Жанры (запятая-разделённые)
- `publisher` — Издатель
- `star_rating` — Рейтинг (средняя оценка)
- `age_rating` — Возрастной рейтинг
- `in_ps_plus` — Включено ли в PS Plus
- `voice_langs` — Доступные языки озвучки
- `screen_langs` — Доступные языки субтитров
- `offline_players` — Количество локальных игроков

## Обработка ошибок

Если при сборе сервер ограничивает доступ по IP:

1. **Используйте прокси** (если доступно)
2. **Увеличьте паузу между запросами:**
   ```bash
   uv run src/get_details.py --country en-pl --retry-missing --sleep-seconds 2.0
   ```
3. **Продолжите сбор в другое время** — данные сохраняются после каждой игры

## Заметки

- Скрипт сохраняет прогресс после каждой игры, поэтому его можно безопасно прерывать (Ctrl+C)
- При повторном запуске обработанные игры пропускаются (для каждого региона отдельно)
- Режим `--retry-missing` полезен для переполучения данных из-за ошибок сервера
- Используйте `--sleep-seconds` для соблюдения ограничений по количеству запросов
